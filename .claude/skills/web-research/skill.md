# Web Research — 체계적 웹 리서치 스킬

## 목적
주어진 주제에 대해 체계적으로 웹 검색을 수행하고, 신뢰할 수 있는 정보를 수집/정리하는 스킬.

## 워크플로우

### Step 1: 검색 전략 수립
1. 주제에서 핵심 키워드 3-5개 추출
2. 영문/한글 키워드 모두 준비
3. 검색 쿼리 조합 생성 (일반, 최신, 기술적)

### Step 2: 다단계 검색
1. **1차 검색**: 핵심 키워드로 개요 파악
   - WebSearch로 상위 결과 확인
   - 공식 출처 (공식 블로그, 논문, 보도자료) 우선 식별

2. **2차 검색**: 구체적 사실 확인
   - 날짜, 수치, 버전 등 구체적 정보 검색
   - 특정 출처 사이트 지정 검색 (site:openai.com 등)

3. **3차 검색**: 심화 및 교차 확인
   - 다른 관점의 분석 기사/블로그 검색
   - 비교/평가 자료 검색

### Step 3: 출처 접근 및 검증
- WebFetch로 주요 출처 원문 확인
- 정보의 최신성 확인 (발행일 체크)
- 1차 출처 vs 2차 출처 구분

### Step 4: 정보 정리
- 사실(fact)과 의견(opinion) 구분
- 출처별 신뢰도 등급 부여
- 구조화된 형태로 정리

## 출처 신뢰도 기준
1. **최고**: 공식 블로그, 논문 (arxiv), 공식 문서
2. **높음**: 주요 기술 매체 (TechCrunch, The Verge, Ars Technica)
3. **보통**: 전문 블로그, 분석 리포트
4. **참고**: 개인 블로그, 소셜 미디어, 포럼

## 출력 형식
```markdown
## [주제] 리서치 결과

### 핵심 사실
- [사실 1] — 출처: [URL] (신뢰도: 높음)
- [사실 2] — 출처: [URL] (신뢰도: 최고)

### 상세 내용
[구조화된 내용]

### 출처 목록
1. [제목] - [URL] - [발행일] - [신뢰도]
```

## 도구 사용법
- **WebSearch**: `query` 파라미터에 검색어 입력. 영문 쿼리가 더 풍부한 결과 제공
- **WebFetch**: `url` 파라미터에 URL 입력. 긴 페이지는 핵심 부분만 추출
